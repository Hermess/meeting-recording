# 智能妙记用户操作手册

适用版本：AI 会议纪要视觉报告生成器当前本地 Web 版  
更新日期：2026-05-04

## 1. 产品说明

智能妙记用于把会议录音、会议转写文本或已有会议材料，整理成可编辑的会议纪要、会议总结长图、Word 文档，并支持发布到语雀知识库。

典型流程：

```text
登录账号
  -> 完成个人配置
  -> 新建会议
  -> 录音转写 / 上传粘贴材料
  -> 确认原始内容
  -> 生成 AI 纪要和总结长图
  -> 编辑纪要正文
  -> 下载 Word / 下载长图 / 推送语雀
```

## 2. 登录与账号

### 2.1 登录

1. 打开系统地址：`http://localhost:3000`
2. 未登录时，系统会自动跳转到登录页。
3. 输入邮箱和密码，点击「登录」。

### 2.2 注册账号

1. 在登录页点击「注册账号」。
2. 输入邮箱，点击「获取验证码」。
3. 输入邮箱验证码、密码和姓名。
4. 点击「注册并登录」。

说明：本地开发环境下，如果邮箱服务未配置，页面可能展示开发验证码，可直接复制使用。

### 2.3 忘记密码

1. 在登录页点击「忘记密码」。
2. 输入邮箱并获取验证码。
3. 验证通过后设置新密码。
4. 使用新密码重新登录。

### 2.4 数据隔离

登录后，会议、模型配置、ASR 配置、语雀配置和个人热词均按当前账号隔离。用户只能查看和处理自己账号下的会议记录。

## 3. 首次使用配置

进入「设置」页后，建议按下面顺序完成配置。

### 3.1 模型网关

模型网关用于生成会议纪要、同步 Markdown 修改后的结构化数据，并参与总结长图内容生成。

操作步骤：

1. 打开「设置」。
2. 进入「模型网关」配置项。
3. 点击「新增模型网关配置」。
4. 填写配置名称、模型网关 URL、API Key、模型名称、温度、最大 Token 等参数。
5. 点击「测试模型」。
6. 测试通过后，才能保存。
7. 需要作为默认纪要模型时，勾选「设为默认」。

字段说明：

| 字段 | 含义 |
| --- | --- |
| 配置名称 | 给业务用户看的名称，例如「MiniMax 纪要模型」 |
| 模型网关 URL | 模型接口地址，通常是兼容 OpenAI Chat Completions 的地址 |
| API Key | 模型网关访问密钥 |
| 模型名称 | 实际调用的模型 ID，例如 `MiniMax-M2.7` |
| 设为默认 | 新建会议时默认使用该模型 |

注意：新建会议时只选择已保存的模型配置名称，不需要重新填写模型参数。

### 3.2 实时 ASR

实时 ASR 用于浏览器录音时把语音实时转成文字。

操作步骤：

1. 打开「设置」。
2. 进入「实时 ASR」配置项。
3. 填写豆包/火山 ASR 的 App ID、Access Token、Secret Key、Cluster / Resource ID 等参数。
4. 如已在火山控制台配置替换词表，可填写替换词 ID。
5. 点击「测试 ASR」。
6. 测试通过后保存。

说明：

- 浏览器录音会把音频转为 16k PCM，并按约 200ms 分包发送给后端。
- ASR 凭证由每个用户自行配置，互不共享。
- 如果 ASR 失败，可以使用「上传 / 粘贴材料」继续完成纪要生成。

### 3.3 语雀发布

语雀配置用于把会议纪要发布到指定知识库。

操作步骤：

1. 打开「设置」。
2. 进入「语雀发布」配置项。
3. 填写语雀 Token。
4. 点击「测试并读取知识库」。
5. 测试通过后，系统会读取当前账号可访问的知识库列表。
6. 点击「保存语雀」。

说明：

- 新建会议时可以选择默认发布知识库。
- 纪要页也可以在发布前重新选择知识库。
- 发布内容包含纪要信息、总结长图和纪要正文。
- 如勾选「随文档发布录音」，小录音片段会以内嵌文件方式写入正文；超过系统限制的大录音会提示暂不能随文档发布。

### 3.4 个人热词

个人热词用于帮助纪要模型更准确识别人名、项目名、系统名和专业术语。

操作步骤：

1. 打开「设置」。
2. 进入「个人热词」。
3. 点击「添加热词」。
4. 输入词汇，并选择类型。
5. 点击保存。

支持的热词类型包括：

- 人名
- 项目名
- 系统名
- 专业术语
- 其他

说明：

- 个人热词不限制个数。
- 当前个人热词主要用于纪要生成阶段，帮助模型理解转写内容。
- 实时 ASR 阶段如果需要直接优化语音识别结果，需要同时配置火山 ASR 的替换词 ID。

## 4. 新建会议

从首页点击「新建会议」进入创建页。

需要填写：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 会议名称 | 是 | 用于会议列表、纪要标题和导出文件名 |
| 参会人员 | 否 | 多人可用顿号、逗号或换行分隔 |
| 默认纪要模型 | 自动 | 从设置页默认模型读取 |
| 总结图模板 | 自动 | 默认使用会议总结长图模板 |
| 语雀知识库 | 否 | 发布语雀时的默认知识库 |
| 语雀公开级别 | 否 | 可继承知识库默认、公开或私密 |

## 5. 两种会议输入方式

新建会议时需要选择一种输入方式。

### 5.1 录音转写

适用于正在开的现场会议。

流程：

```text
选择「录音转写」
  -> 创建并进入录音页
  -> 点击「开始录音并转写」
  -> 允许浏览器麦克风权限
  -> 实时查看转写内容
  -> 可暂停转写
  -> 点击「结束会议」
  -> 生成会议纪要
```

注意：

- 录音不会一进入页面就自动开始，需要手动点击「开始录音并转写」。
- 只有点击「结束会议」后，「生成会议纪要」按钮才会可用。
- 暂停只停止实时转写，后续可以继续处理已经保存的转写内容。
- 录音会按分片保存，避免长会议占用过多内存。

### 5.2 上传 / 粘贴材料

适用于已经有会议转写、会议草稿、Markdown、Word 或 PDF 材料的场景。

流程：

```text
点击首页「上传 / 粘贴材料」
  -> 默认进入上传 / 粘贴模式
  -> 粘贴文本或上传附件
  -> 系统提取原始内容
  -> 确认转写内容
  -> 点击「生成会议纪要」
```

支持的附件类型：

- `.md`
- `.doc`
- `.docx`
- `.pdf`

说明：

- 上传 / 粘贴模式不显示录音控件。
- 原始内容确认页不建议直接编辑原始转写。
- 如需修改正式纪要，请进入纪要页编辑 Markdown 正文。

## 6. 会议列表

首页展示所有会议文件。

列表字段包括：

- 文件名称
- 状态
- 转写段落数
- 行动项数量
- 最近更新时间
- 操作

常见操作：

| 状态 | 操作按钮 | 含义 |
| --- | --- | --- |
| 草稿 / 录音中 | 继续录音 / 继续导入 | 回到转写或导入页继续处理 |
| 待生成 / 失败 | 生成纪要 | 进入纪要页并生成 AI 纪要 |
| 已生成 / 待发布 / 已发布 | 查看纪要 | 查看、编辑、下载或发布 |

## 7. 纪要页

纪要页是最终处理会议内容的主要页面。

### 7.1 页面结构

纪要页一般包含：

- 会议录音或上传材料说明
- 原始转写内容
- AI 纪要
- 章节
- 总结长图预览
- 纪要正文
- 更多操作

如果是上传 / 粘贴材料创建的会议，页面会提示该会议没有录音播放控制。如需更换原始内容，可返回导入页重新上传或粘贴。

### 7.2 生成纪要

点击「生成纪要」后，系统会调用当前选择的模型配置生成：

- 会议背景
- 会议总结
- 核心里程碑
- 风险与关注点
- 关键行动项
- AI 洞察
- 章节
- Markdown 纪要正文
- 总结长图数据

如果对生成结果不满意，可以在顶部切换另一个已保存的模型配置，然后点击「重新生成」。

### 7.3 总结长图

生成纪要后，系统会自动生成总结长图。

使用方式：

- 在纪要页直接预览。
- 点击图片可放大查看。
- 图片区域固定高度，支持上下滚动查看完整长图。
- 在「更多」中可下载总结长图 PNG。

说明：

- 总结长图只展示图示报告部分，不包含页面搜索框或外层操作区。
- 如果纪要内容质量未通过审核，系统会阻止生成或发布，避免输出低质量长图。

### 7.4 编辑纪要正文

纪要正文使用 Markdown 方式编辑。

操作步骤：

1. 在纪要页点击「编辑正文」或「更多」中的「编辑纪要」。
2. 修改 Markdown 正文。
3. 点击「保存并刷新」。
4. 系统会重新同步结构化数据，并刷新总结长图。

注意：

- 未保存并刷新时，下载 Word、下载长图、发布语雀等下游操作会被阻止。
- 非编辑状态下，Markdown 会渲染成带标题、列表、待办、表格的阅读样式。

## 8. 下载与发布

在纪要页点击「更多」可进行后续操作。

### 8.1 下载总结 Word

下载内容包括：

- 纪要标题
- 纪要信息
- 总结长图
- 纪要正文
- 行动项表格

### 8.2 下载总结长图

下载当前会议的 PNG 总结长图。

### 8.3 推送语雀

操作步骤：

1. 点击「更多」。
2. 选择语雀知识库。
3. 选择公开级别。
4. 如需同时发布录音，勾选「随文档发布录音」。
5. 点击「推送语雀」。

发布成功后，会议列表会展示语雀入口。

发布内容：

- 纪要信息
- 总结长图
- 会议录音，需勾选且满足大小限制
- 纪要正文

录音说明：

- 当前语雀发布支持把小录音片段以内嵌文件方式写入正文。
- 超过 20MB 的录音片段不会直接发布，系统会提示需要接入语雀附件上传、OAuth 或对象存储。
- 长会议建议优先发布纪要正文和总结长图，录音文件后续可通过专门的文件存储方案归档。

## 9. 推荐使用流程

### 9.1 现场会议

```text
新建会议
  -> 选择录音转写
  -> 创建并进入录音页
  -> 手动开始录音并转写
  -> 会议中观察转写是否正常
  -> 结束会议
  -> 生成纪要
  -> 检查总结长图
  -> 编辑正文
  -> 保存并刷新
  -> 下载 Word 或推送语雀
```

### 9.2 已有会议材料

```text
首页点击上传 / 粘贴材料
  -> 粘贴正文或上传附件
  -> 确认提取出的原始内容
  -> 生成会议纪要
  -> 检查 AI 纪要和总结长图
  -> 编辑正文
  -> 保存并刷新
  -> 下载 Word 或推送语雀
```

## 10. 常见问题

### 10.1 为什么进业务页面会跳到登录页？

系统要求登录后才能访问会议、设置和发布功能。请先完成邮箱登录或注册。

### 10.2 为什么无法生成纪要？

常见原因：

- 没有录音转写或上传 / 粘贴任何内容。
- 录音会议尚未点击「结束会议」。
- 没有配置默认纪要模型。
- 模型网关测试未通过。

### 10.3 为什么 ASR 有音频包但没有文字？

可能原因：

- 麦克风设备选错。
- 当前输入音量过低。
- ASR 凭证不正确。
- 火山 ASR 服务返回异常。
- 浏览器权限没有正常授权。

建议：

1. 在录音页刷新麦克风列表。
2. 选择明确的麦克风设备。
3. 观察当前音量和峰值音量。
4. 到设置页重新测试 ASR。
5. 如仍不可用，使用上传 / 粘贴材料兜底。

### 10.4 为什么生成的长图质量差？

常见原因：

- 原始转写本身不完整。
- 模型没有正确理解会议结构。
- 使用了兜底纪要，而不是模型生成纪要。
- Markdown 修改后没有点击「保存并刷新」。

建议：

1. 检查原始转写是否完整。
2. 切换更合适的模型配置重新生成。
3. 在 Markdown 正文中补齐关键内容。
4. 点击「保存并刷新」重新生成结构化数据和长图。

### 10.5 为什么语雀发布成功但知识库里看不到？

可能原因：

- 发布到了另一个知识库。
- 当前知识库目录未刷新。
- 语雀接口创建了文档，但目录插入失败。
- 文档公开级别或权限设置导致不可见。

建议：

1. 从系统里的语雀链接直接打开文档。
2. 检查发布时选择的知识库。
3. 刷新语雀知识库目录。
4. 如仍不可见，重新发布一次。

### 10.6 个人热词和 ASR 替换词有什么区别？

| 类型 | 作用阶段 | 用途 |
| --- | --- | --- |
| 个人热词 | 纪要生成阶段 | 帮助模型理解人名、项目名、系统名、专业术语 |
| ASR 替换词 ID | 实时语音识别阶段 | 帮助豆包/火山 ASR 更准确识别语音中的专有名词 |

两者可以同时使用。

## 11. 注意事项

- 会议原始转写不建议直接编辑，正式内容请在纪要正文中编辑。
- 录音会议需要先结束会议，再生成纪要。
- 修改纪要正文后，需要点击「保存并刷新」。
- 发布语雀前，请确认知识库、公开级别和是否包含录音。
- 长会议录音可能较大，建议谨慎勾选随文档发布录音。
- 如模型输出质量不理想，优先切换模型重新生成，再进行人工编辑。
